## ğŸ¯ Component Responsibilities

### Frontend (Next.js/React)
- **Responsibility**: User interface and experience
- **Key Components**:
  - `get-started.tsx` - Onboarding flow (redirects to settings after bootstrap)
  - `settings.tsx` - Data ingestion dashboard and symbol management
  - `DataIngestionDashboard.tsx` - Live status display component
- **Interactions**:
  - Calls API endpoints
  - Polls for status updates (every 2 seconds)
  - Displays notifications
  - Handles URL params for job tracking

### API Layer (FastAPI)
- **Responsibility**: Business logic and orchestration
- **Key Modules**:
  - `api/routes/admin.py` - Bootstrap & admin endpoints
  - `api/routes/data_ingestion.py` - Ingestion management
- **Interactions**:
  - Receives HTTP requests
  - Creates job records in MongoDB
  - Enqueues Celery tasks
  - Returns status information

### Task Queue (Celery + Redis)
- **Responsibility**: Asynchronous task execution
- **Key Components**:
  - Celery workers (Python processes)
  - Redis broker (message queue)
  - Beat scheduler (cron-like)
- **Interactions**:
  - Receives tasks from API
  - Distributes to available workers
  - Tracks task state

### Workers (Celery)
- **Responsibility**: Data processing and ingestion
- **Key Tasks**:
  - `ingest_symbol_interval_task` - Fetch & store data
  - `batch_ingest_task` - Orchestrate multiple ingestions
  - `backfill_gaps_task` - Fill missing data
- **Interactions**:
  - Fetches from exchange APIs
  - Writes to MongoDB
  - Updates job status
  - Sends notifications

### Database (MongoDB)
- **Responsibility**: Persistent data storage
- **Key Collections**:
  - `symbols` - Symbol metadata and status
  - `ingestion_jobs` - Job tracking and progress
  - `ohlcv` - Raw price data
  - `features` - Technical indicators
  - `notifications` - User notifications
- **Interactions**:
  - Read/write from all components
  - Indexed for fast queries
  - Aggregation pipelines for reporting

### External Services
- **Responsibility**: Market data provision
- **Services**:
  - Binance API (via CCXT)
  - Other exchanges as needed
- **Interactions**:
  - Rate-limited requests
  - OHLCV data retrieval
  - Historical data backfill
